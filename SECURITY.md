# 安全政策

## 支持的版本

| 版本 | 安全维护状态 |
| --- | --- |
| `main` 分支最新提交 | ✅ 维护中 |

FEK 目前处于开源演示 / 黑客松阶段，尚无正式发行版本。安全修复会直接在 `main` 分支提交。

## 报告安全漏洞

**请勿通过公开 Issue 报告安全漏洞。**

请通过以下任一方式私下提交：

- 在 GitHub 仓库页面发起 **Private vulnerability report（私有安全通告）**；
- 或发送邮件至安全联系人：**fek-security@example.com**（请替换为实际维护邮箱）。

报告中请尽量包含：

- 漏洞类型与影响范围；
- 复现步骤（最小可复现示例最佳）；
- 受影响环境（操作系统 / Python 版本 / FEK 提交哈希）。

## 响应时间

- **首次响应**：收到报告后 72 小时内；
- **修复与披露**：确认后尽快修复，具体节奏视严重程度而定；
- 修复发布前，我们会与你协调披露时间，避免 0-day 扩散。

## 使用须知

- FEK 默认运行在 **mock 模式**：不调用任何外部大模型、不向外发送数据、无需任何 API 密钥，适合本地安全评估。
- 若启用真实 LLM 模式（设置环境变量 `FEK_MODE=openai`），请务必在本地妥善保管你的 `OPENAI_API_KEY`，**不要**将其提交到仓库、Issue 或任何日志中。`.env` 已被 `.gitignore` 排除。
- 遥测数据（执行成本 / 延迟 / 质量）默认仅写入本地文件 `fek_traces.jsonl`，不会自动上传。

## 免责声明

FEK 为研究 / 演示性质的执行内核原型，不作为生产级安全组件使用。在生产环境部署前，请自行完成安全评审。
